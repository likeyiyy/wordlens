# 《中文反义词与认知透镜》网站项目计划书

## 一、项目核心思想

人类认识世界的起点是二元对立：高低、上下、冷热、多少、明暗等反义词，是最朴素的认知框架，也是词语里凝结的底层思考逻辑。

二元认知会被误解为「二极管思维」，现实存在灰色边界，既不属于此也不属于彼，这是认知的关键挑战。

认知会从「二分」进化为多维透镜：二维象限、九宫格等模型，是观察世界的更精细工具。

中文基础字 / 词量级有限（一级 + 二级 + 三级汉字约万字内），适合做轻量化、结构化展示。

**项目本质**：做一个「文字版认知透镜工具」，用网站呈现：反义词（二元起点）→ 双字词（拓展）→ 灰度边界 → 多维认知模型，把抽象认知思路可视化。

---

## 二、项目定位与目标

### 1. 定位

轻量、静态、可快速访问的中文基础认知词库网站，以「反义词」为核心，展示人类从二元到多维的认知路径。

### 2. 目标

- 完成 Next.js 项目开发，部署上线到 Vercel
- 展示一套精简、规范的中文基础反义词库
- 展示反义词拓展而来的常用双字词
- 简单可视化：二元对立 → 灰色地带 → 二维 / 九宫格认知透镜
- 界面简洁、打开快、移动端 / PC 端适配

---

## 三、核心功能模块

### 1. 基础反义词展示模块（核心）

按维度分类：

- **空间类**：上下、高低、前后、内外、远近
- **状态类**：冷热、明暗、动静、干湿、软硬
- **程度类**：多少、轻重、快慢、大小、长短
- **逻辑类**：是非、对错、真假、有无

形式：卡片 / 列表展示，一对一对齐展示。

### 2. 双字词拓展模块

基于单字反义词，拓展常用双字词：

- 例：高→高级 / 高尚，低→低级 / 低下
- 例：冷→寒冷 / 冷淡，热→炎热 / 热情

与原反义词关联展示。

### 3. 认知透镜示意模块（可视化）

- **二元区**：清晰对立的词
- **灰度区**：模糊、中间态词语（如：温、中等、普通、适中）
- **二维象限图 / 九宫格简图**：将词语按两个维度映射，体现「非二极管」的多维视角

### 4. 基础交互（极简）

- 搜索：搜汉字 / 词语，快速定位反义词
- 分类切换：按空间 / 状态 / 程度等筛选
- 视图切换：列表视图 / 透镜视图

### 5. 词库数据

使用静态 JSON 文件存储：

- `antonyms.json` - 反义词表
- `compound-words.json` - 双字词表
- `gradient-words.json` - 灰度中间词表
- `dimensions.json` - 认知维度映射表

不依赖数据库，方便 Vercel 部署。

## 六、数据内容规划

### 反义词分类（精简版：约 50 对）

**空间类**：上下、高低、前后、内外、远近、左右、东西、南北、深浅、宽窄

**状态类**：冷热、明暗、动静、干湿、软硬、生死、开关、新旧、好坏、强弱

**程度类**：多少、轻重、快慢、大小、长短、粗细、厚薄、浓淡、繁简

**逻辑类**：是非、对错、真假、有无、黑白、阴阳

### 灰度中间词（约 20 个）

中等、普通、一般、适中、温、平和、中立、半、过渡、临界、微、稍、约、朦胧、含蓄...

### 双字词拓展（每对 4-8 个）

- 高 → 高级、高尚、高大、高雅
- 低 → 低级、低下、低矮、低俗
- 冷 → 寒冷、冷淡、冰冷、清凉
- 热 → 炎热、热情、火热、燥热

## 七、UI 设计风格

- **极简主义**：大量留白，聚焦内容
- **响应式**：移动端优先，PC 端适配
- **颜色方案**：
  - 主色：深灰/黑（文字）、白（背景）
  - 强调色：柔和的红蓝对立色（体现二元概念）
- **字体**：系统默认中文字体，保证加载速度
